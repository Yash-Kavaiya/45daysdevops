# 45 Days DevOps — Zero to Hero

A structured DevOps learning repository based on the **DEVOPS ZERO TO HERO COURSE** YouTube playlist by Abhishek Veeramalla.

This repo is organized as a practical companion for beginners who want to learn DevOps step by step: Linux, shell scripting, Git, cloud basics, containers, CI/CD, Docker, Kubernetes, monitoring, and real-world project workflows.

Playlist: https://youtube.com/playlist?list=PLdpzxOOAlwvIKMhk8WhzN1pYoJ1YU8Csa

---

## What You Will Learn

- DevOps fundamentals and SDLC improvement
- Linux basics and shell scripting for automation
- Git, GitHub, branching strategies, and interview preparation
- AWS and Azure virtual machine basics
- Infrastructure as Code with Terraform
- Configuration management with Ansible
- CI/CD concepts with Jenkins and GitHub Actions
- Docker fundamentals, networking, volumes, Compose, and best practices
- Kubernetes architecture, pods, deployments, services, ingress, RBAC, CRDs, ConfigMaps, Secrets, monitoring, and live projects
- Practical DevOps interview-focused knowledge

## Prerequisites

Before starting, install or prepare the following:

- A laptop or desktop with internet access
- Basic command-line familiarity
- Git and GitHub account
- AWS account for cloud practice
- Docker Desktop or Docker Engine
- VS Code or any code editor
- Optional: Azure account, Jenkins, Terraform, Ansible, Kubernetes tools

Useful tools:

```bash

docker --version
kubectl version --client
terraform version
ansible --version
```

## Suggested Study Plan

1. Watch one video per day.
2. Read or create notes for the topic.
3. Run every command yourself instead of only watching.
4. Push your daily notes, scripts, and projects to GitHub.
5. Revisit interview-question videos after completing the practical sections.
6. Build at least one end-to-end project using Git, Docker, CI/CD, cloud, and Kubernetes.

## Hands-On Practice Ideas

- Create and connect to an EC2 instance.
- Automate a Linux task using shell scripting.
- Write a script that integrates with the GitHub API.
- Create a Git branching workflow for a sample project.
- Dockerize a web application.
- Build a Jenkins pipeline.
- Create a GitHub Actions workflow.
- Deploy an application to Kubernetes.
- Configure Kubernetes Services, Ingress, ConfigMaps, and Secrets.
- Set up Prometheus and Grafana monitoring.

## Recommended Commands to Practice

```bash
# Git
git init
git status
git add .
git commit -m "message"
git branch
git checkout -b feature-branch
git merge feature-branch
git push origin main

# Linux
pwd
ls -la
cd /path/to/directory
cat file.txt
chmod +x script.sh
./script.sh

# Docker
docker build -t app:latest .
docker run -p 8080:8080 app:latest
docker ps
docker images
docker compose up -d

# Kubernetes
kubectl get nodes
kubectl get pods
kubectl apply -f manifest.yaml
kubectl describe pod <pod-name>
kubectl logs <pod-name>
kubectl delete -f manifest.yaml
```

## How to Use This Repository

```bash
git clone https://github.com/Yash-Kavaiya/45daysdevops.git
cd 45daysdevops
```

Then open the day-wise folders and follow the notes along with the playlist.

> Note for Windows users: some older file names in this repository may contain characters that can cause checkout issues on Windows. If cloning fails, use Git Bash, WSL, or rename the affected files from GitHub before cloning again.
